# 半生以后清醒课：日常内容工作流

这份文档是账号每天的执行手册。目标不是单次写出一篇文章，而是让每次发布的真实结果，持续影响下一篇的选题、标题、结构和排版。

## 工作流总览

```text
公众号后台数据
        ↓
记录数据与复盘
        ↓
选择今天的选题
        ↓
成稿、标题、封面
        ↓
清醒雅阅排版与校验
        ↓
公众号后台发布
        ↓
次日回收数据，进入下一轮
```

## 当前适用路线

当前公众号没有“图文分析数据接口”权限，因此采用截图录入路线：在公众号后台打开上一期的内容分析页，截取数据后发给 Codex。不要再运行 `scripts/run_daily_retro.py`，直到账号获得该接口权限。

截图中优先包含以下指标：

- 阅读人数
- 平均阅读时长
- 完成率
- 新增关注
- 分享、点赞、收藏、留言

## 每日步骤

### 1. 回收上一期数据

1. 打开公众号后台的“内容分析”。
2. 选中昨天或最近一期文章，截取完整数据页。
3. 在本任务中发送截图，并说“分析数据，制作下一篇”。

Codex 将数据写入以下位置：

```text
published/YYYY-MM-DD-主题.json
retros/YYYY-MM-DD-主题.md
memory/rules.md
```

其中 `published` 保存原始指标，`retros` 保存判断，`memory/rules.md` 保存跨文章可复用的规律。

### 2. 制作今天的文章

基于复盘结论，固定交付：

```text
drafts/YYYY-MM-DD-主题.md
assets/covers/主题.png
layouts/YYYY-MM-DD-主题_排版_清醒雅阅.html
layouts/YYYY-MM-DD-主题_排版_清醒雅阅_预览.html
```

写作标准：

- 目标读者：50 岁以后的人。
- 当前优先方向：晚年夫妻、养老钱、子女边界。
- 正文控制在约 1100 至 1500 字，最多三个分点。
- 开头先给具体生活场景，再给读者收益。
- 结尾固定说明账号主题和下一篇预告，帮助新读者决定是否关注。
- 排版遵循“清醒雅阅”：17px 正文、2.0 行高、白底、深灰文字、低饱和墨绿。

### 3. 发布到公众号后台

1. 打开 `layouts` 中带“预览”的 HTML 文件。
2. 点击页面右上角“复制到公众号”。
3. 在公众号编辑器粘贴正文。
4. 使用 `assets/covers` 中的同名图片上传封面。
5. 填入文章标题和摘要后，预览检查，再发布。

发布后，把发布时间告诉 Codex；它会将发布时间补入发布记录，便于下一次复盘。

### 4. Git 留档

每次完成数据复盘和文章制作后，Codex 会提交并推送到 GitHub。GitHub 仓库是账号的长期档案，包含：

- 文章版本与封面
- 发布数据和复盘
- 逐渐形成的写作规则
- 视觉规范和日常操作流程

不提交 AppSecret、后台登录信息或其他敏感凭据。

## 每周一次的检查

每发布 3 至 5 篇，比较：

- 哪个题材带来更高阅读和分享。
- 哪种标题带来更高点击。
- 哪种结构带来更高完成率。
- 哪篇带来新增关注，以及文章结尾是否清楚表达账号价值。

不要只看阅读量。当前排序是：新增关注、完成率、收藏和分享、阅读量。

## 以后可升级的部分

当公众号取得“图文分析数据接口”权限后，再启用：

```text
python scripts/run_daily_retro.py
```

它会自动拉取昨天的数据并生成初步复盘。具体配置见 [自动获取昨天数据](auto_data_setup.md)。
